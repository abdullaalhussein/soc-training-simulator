Project Overview

نظرة عامة

What

A multi-role SOC training platform that provides realistic simulated security logs for hands-on cybersecurity training — including YARA rule writing, log analysis, and incident response.

Who — Roles

Admin Trainer Trainee
  • Admin — Manage users, scenarios, audit logs, and system settings
  • Trainer — Create sessions, monitor trainees in real-time, adjust scores
  • Trainee — Investigate scenarios, analyze logs, submit evidence & reports

Why

  • Hands-on experience with real-world attack scenarios
  • Multi-stage investigation workflow
  • YARA rule writing and testing in a safe environment
  • Objective multi-dimensional scoring
  • Real-time monitoring and guided training

Tech Stack

TypeScript types, enums (Roles, LogTypes, CheckpointTypes, etc.), validation schemas (Zod), and application constants shared between client & server.

PostgreSQL schema definition with 13 data models, 7 enums, database seed scripts with demo scenarios and users.

Scoring System

نظام التقييم

Score Weights — Total: 100 points

Accuracy 35%
Investigation 20%
Evidence 20%
Response 15%
Report 10%

Accuracy — 35%

Checkpoint answers: True/False, Multiple Choice, Severity Classification. Full or zero points per answer based on correctness.

Investigation — 20%

Search query diversity (5 pts), filter usage (5 pts), log analysis depth (5 pts), timeline/process building (5 pts).

Evidence — 20%

F1 score of selected evidence vs correct evidence. Precision × Recall balanced. Correct if F1 ≥ 0.8.

Response — 15%

Recommended action checkpoints. Evaluates incident response decisions and containment/remediation choices.

Report — 10%

Incident report quality: keyword coverage (50%) + recommendation count (50%). Correct if total ≥ 0.6.

Adjustments

Hint penalty: -5 pts per hint used.
Trainer adjustment: Manual bonus/deduction with notes.

total = accuracy + investigation + evidence + response + report - hintPenalty + trainerAdj

Event Flows

Hint Delivery

Trainer send-hint ⟶ Server validates ⟶ hint-sent to trainee attempt room ⟶ UI displays hint notification

Live Progress Monitoring

Trainee actions ⟶ progress-update ⟶ Server forwards to trainer session room ⟶ Real-time dashboard update

Session Discussion

User send-session-message ⟶ Persisted to DB ⟶ Broadcast to both namespaces in session room

Session Control

Trainer pause-session ⟶ DB status update ⟶ session-paused to all trainees ⟶ UI blocks investigation

Broadcast Alerts

Trainer send-session-alert ⟶ All trainees in session receive notification overlay

Deployment Guide

دليل النشر

Docker Multi-Stage Builds

Server Dockerfile
# Stage 1: Build FROM node:20-alpine AS build RUN npm ci && npx tsc # Stage 2: Production FROM node:20-alpine RUN apk add --no-cache yara COPY --from=build dist/ prisma/ RUN npx prisma db push --skip-generate CMD ["node", "dist/index.js"]
Client Dockerfile
# Stage 1: Build FROM node:20-alpine AS build ARG NEXT_PUBLIC_API_URL NEXT_PUBLIC_WS_URL RUN npm run build # Stage 2: Production FROM node:20-alpine COPY --from=build .next/standalone/ CMD ["node", "client/server.js"]

Railway.app Deployment

3 services on Railway:

PostgreSQL — Railway plugin, auto-provisions DATABASE_URL
Server — Express 5 + Socket.io, uses server/Dockerfile
Client — Next.js standalone, uses client/Dockerfile

CORS auto-allows *.railway.app domains. Railway assigns ports automatically.

Local Development

# Start PostgreSQL + pgAdmin (docker-compose.yml) $ docker-compose up -d # Services running: # PostgreSQL → localhost:5433 # pgAdmin → localhost:5050 # Start dev servers (package.json) $ npm run dev # Client → localhost:3000 # Server → localhost:3001

Implementation Guide

دليل التنفيذ

Local Setup

# 1. Clone & install $ git clone https://github.com/abdullaalhussein/soc-training-simulator.git $ cd soc-training-simulator $ npm install # 2. Copy env & start database (docker-compose.yml) $ cp .env.example .env $ docker-compose up -d # 3. Push schema (prisma/schema.prisma) & seed (prisma/seed.ts) $ npm run db:push $ npm run db:seed # 4. Start development (package.json scripts) $ npm run dev # Client → localhost:3000 Server → localhost:3001

Adding a New Scenario

  • Option A: Use Admin UI — navigate to /scenarios and use the form
  • Option B: JSON import — POST to /api/scenarios/import
    Route handler: server/src/routes/scenarios.ts
  • JSON format: name, difficulty, category, MITRE ATT&CK IDs, briefing, stages[] with nested logs[] & hints[], top-level checkpoints[]

Adding a New Checkpoint Type

1 Add enum to CheckpointType in prisma/schema.prisma
2 Add grading logic in server/src/services/scoring.service.ts gradeAnswer()
3 Add scoring category in scoring.service.ts recalculateScores()
4 Update shared enum in shared/src/types/index.ts
5 Create client component in client/src/components/scenario-player/

Adding a New Log Type

1 Add enum to LogType in prisma/schema.prisma
2 Update shared enum in shared/src/types/index.ts
3 Add display component in client/src/components/scenario-player/LogFeedViewer/
4 Add sample data in prisma/seed.ts
5 Run npm run db:push and npm run db:seed
